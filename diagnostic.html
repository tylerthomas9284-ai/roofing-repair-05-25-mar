🔍 How to Debug:

  1. Open Browser DevTools:
    • Right-click anywhere → "Inspect" or "Inspect Element"
    • Or press F12 (Windows) / Cmd+Option+I (Mac)
  2. Check Console Tab:
    • Look for red errors
    • If you see "Failed to load resource" for main.css, the CSS path is wrong
  3. Check Network Tab:
    • Refresh the page (F5)
    • Find main.css in the list
    • Status should be 200 (green) = Success
    • Status 404 (red) = File not found - check file structure!
  4. Check Elements Tab:
    • Click on any element (like the header)
    • Look at "Styles" panel on the right
    • You should see CSS rules from main.css
    • If you only see "user agent stylesheet", CSS isn't loading

⚠️ Common Issues & Solutions:

Issue 1: CSS Not Loading

Symptoms: No colors, everything is black text on white background

Solution:

  • Verify styles/main.css exists in the same folder as your HTML
  • Check HTML has: <link rel="stylesheet" href="styles/main.css">
  • NOT: href="/styles/main.css" (no leading slash)

Issue 2: Opening from File Explorer

Symptoms: Works on some pages but not others

Solution: Always open index.html first, then use navigation links

Issue 3: Browser Cache

Symptoms: Fixes don't seem to apply

Solution: Hard refresh:

  • Windows: Ctrl + Shift + R or Ctrl + F5
  • Mac: Cmd + Shift + R

Issue 4: Wrong File Structure

Symptoms: 404 errors in console

Solution: Move files to correct locations as shown in "File Structure Check" above

💡 Pro Tips:

  • Use a local server: Instead of opening files directly, run: python -m http.server 8000 in the public/ folder, then visit http://localhost:8000
  • Try different browsers: If one doesn't work, try Chrome, Firefox, or Edge
  • Check permissions: Make sure you have read permission on all files
  • Disable extensions: Some browser extensions can interfere with local files
